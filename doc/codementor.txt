1. INTRODUCTION                                         *codementor-introduction*

CodeMentor is a Neovim plugin that acts as your personal coding mentor. Unlike
code generation tools like GitHub Copilot, CodeMentor explains concepts and
guides your learning instead of writing code for you.

Philosophy: "Teach a person to fish, don't fish for them."

==============================================================================
2. REQUIREMENTS                                         *codementor-requirements*

- Neovim 0.9.0+
- plenary.nvim (for HTTP requests)
- Tree-sitter (for context extraction)
- An API key for Claude/OpenAI (or local Ollama)

==============================================================================
3. INSTALLATION                                         *codementor-installation*

Using lazy.nvim: >lua
    {
      "your-username/codementor",
      dependencies = {
        "nvim-lua/plenary.nvim",
        "nvim-treesitter/nvim-treesitter",
      },
      config = function()
        require("codementor").setup({
          provider = "claude",
          api_key = os.getenv("ANTHROPIC_API_KEY"),
        })
      end,
    }
<

==============================================================================
4. CONFIGURATION                                       *codementor-configuration*

Default configuration: >lua
    require("codementor").setup({
      -- LLM Provider: "claude" | "openai" | "ollama"
      provider = "claude",
      model = "claude-sonnet-4-20250514",

      -- Behavior
      default_mode = "question",
      context_lines = 50,
      include_imports = true,
      language = "English",

      -- UI
      ui = {
        width = 80,
        height = 20,
        border = "rounded",
        max_width = 120,
      },

      -- Keymaps
      keymaps = {
        ask = "<leader>ma",
        close = "q",
        copy = "y",
      },
    })
<

==============================================================================
5. USAGE                                                       *codementor-usage*

1. Write a comment with a mentor prefix: >
    // Q: What is the time complexity of this algorithm?
<

2. Press <leader>ma (or your configured keymap)

3. Read the explanation in the floating window

4. Press 'q' to close, 'y' to copy

==============================================================================
6. MODES                                                       *codementor-modes*

CodeMentor supports 5 interaction modes:

*Q:* Question Mode                                          *codementor-question*
    Direct answers with explanations
    Example: // Q: What does this regex do?

*S:* Socratic Mode                                          *codementor-socratic*
    Guided discovery through questions
    Example: // S: Why might a mutex be better here?

*R:* Review Mode                                              *codementor-review*
    Code review with security, performance, best practices
    Example: // R: Review this function

*D:* Debug Mode                                                *codementor-debug*
    Guided debugging process
    Example: // D: This returns nil sometimes, why?

*E:* Explain Mode                                            *codementor-explain*
    Deep concept explanations
    Example: // E: Explain closures in JavaScript

==============================================================================
7. COMMANDS                                                 *codementor-commands*

*:CodeMentor* [subcommand]
    Main command with optional subcommand:
    - ask       Trigger on current mentor comment (default)
    - question  Force Question mode
    - socratic  Force Socratic mode
    - review    Force Review mode
    - debug     Force Debug mode
    - explain   Force Explain mode
    - modes     Show available modes help
    - close     Close popup window
    - version   Show version

*:CodeMentorAsk*      Same as :CodeMentor ask
*:CodeMentorQuestion* Force Question mode
*:CodeMentorSocratic* Force Socratic mode
*:CodeMentorReview*   Force Review mode
*:CodeMentorDebug*    Force Debug mode
*:CodeMentorExplain*  Force Explain mode
*:CodeMentorClose*    Close popup
*:CodeMentorModes*    Show modes help

==============================================================================
8. KEYMAPS                                                   *codementor-keymaps*

Default keymaps (configurable):

Normal mode:
    <leader>ma    Ask mentor about current comment

Popup window:
    q             Close popup
    <Esc>         Close popup
    y             Copy content to clipboard
    j/k           Navigate (with wrap)
    <C-d>/<C-u>   Scroll half page

==============================================================================
9. API                                                           *codementor-api*

Lua API: >lua
    local codementor = require("codementor")

    -- Setup
    codementor.setup(opts)

    -- Ask about current comment
    codementor.ask()

    -- Ask with specific mode
    codementor.ask_mode("question")
    codementor.ask_mode("socratic")
    codementor.ask_mode("review")
    codementor.ask_mode("debug")
    codementor.ask_mode("explain")

    -- Show modes help
    codementor.show_modes()

    -- Get version
    codementor.version()
<

UI API: >lua
    local ui = require("codementor.ui")

    ui.show(content, mode, question)
    ui.close()
    ui.is_open()
<
